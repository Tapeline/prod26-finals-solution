# Глоссарий единого языка

## Доступ

**access:Администратор**
:   (_Administrator_)

    Пользователь, обладающий полномочиями управления системой 
    (создание/изменение пользователей, назначение ролей, управление
    правилами утверждения).


**access:Экспериментатор**
:   (_Experimenter_)

    Пользователь, управляющий экспериментами.


**access:Утверждающий**
:   (_Approver_)

    Пользователь, утверждающий эксперименты.


**access:Наблюдатель**
:   (_Viewer_)

    Пользователь, имеющий право только на просмотр информации в системе.


**access:ID**
:   Строка (UUID), уникальная для каждого пользователя и 
    идентифицирующая его в системе.


**access:Группа Утверждающих**
:   Группа из <access:Утверждающий|Утверждающих> и минимальное 
    количество их согласий, необходимое для принятия решения.


**access:Роль**
:   Одна из четырёх ролей: 

    - <access:Администратор>
    - <access:Экспериментатор>
    - <access:Утверждающий>
    - <access:Наблюдатель>

## Эксперименты


**exp:Feature flag**
:   Параметры поведения продукта, которые можно переключать без нового релиза. 
    Флаг состоит из ключа и значения по умолчанию, которое используется, 
    если эксперимент не применяется.


**exp:Ключ флага**
:   Уникальный идентификатор флага


**exp:Тип флага**
:   Один из трёх возможных типов значений флага:

    - строка (string)
    - число (number)
    - логическое значение (boolean)


**exp:ID**
:   Уникальный идентификатор эксперимента


**exp:Эксперимент**
:   Временная конфигурация, которая подменяет значение конкретного feature 
    flag для части пользователей с целью проверки гипотез. 
    Эксперимент содержит варианты, таргетинг, метрики, guardrails и 
    проходит через жизненный цикл от черновика до завершения.


**exp:Статус**
:   Одно из возможных состояний эксперимента:

    - Черновик — редактируется.
    - На ревью — ждёт ревью.
    - Одобрен — прошёл ревью, готов к запуску.
    - Запущен — активен, участвует в выдаче значений флага.
    - На паузе — временно остановлен (не выдаёт значение как эксперимент), но сохраняет конфигурацию.
    - Завершён — решение принято.
    - В архиве — для истории.
    - Отклонён — может быть переделан и отправлен заново.


**exp:Версия**
:   Номер версии конфигурации эксперимента, начиная с 1.


**exp:Вариант**
:   Вариант эксперимента — значение, подставляемое попавшему в эксперимент
    пользователю, вместо значения по умолчанию.


**exp:Правило таргетирования**
:   Описанное на формальном языке правило фильтрации пользователей,
    попадающих в эксперимент.


**exp:Итог эксперимента**
:   Один из возможных вариантов с текстовым комментарием:

    - Раскатка победителя (winner rollout)
    - Откат к значению по умолчанию
    - Нет эффекта


**exp:Метрика**
:   Одна из возможных типов метрик эксперимента:

    - целевая — основная метрика, по которой команда оценивает успех эксперимента.
    - вспомогательная
    - аварийная — метрики, по которым выполняются автоматические остановки/откаты.

    Ссылается на <stat:ID Метрики>.


## Выдача


**rt:Субъект**
:   Некто (пользователь или устройство), запрашивающий конфигурацию Feature flags.

**rt:ID Субъекта**
:   Стабильный идентификатор субъекта (пользователя или устройства), для которого 
    принимается решение.


**rt:Атрибуты субъекта**
:   Набор характеристик (страна, версия приложения, платформа), 
    передаваемых в момент запроса для работы таргетинга.


**rt:Решение**
:   (_Decision_)

    Конкретное решение о выдаче субъекту тех или иных значений флагов.


**rt:ID Решения**
:   Уникальный идентификатор конкретного решения о выдаче флага. 
    Связывает показ варианта с последующими событиями.


**evt:ID**
:   Уникальный идентификатор события.


## События


**evt:Событие**
:   (_Event_)
    Факт взаимодействия пользователя с продуктом (клик, показ, покупка).
    Обязательно принадлежит одному из Типов событий.


**evt:ID Типа**
:   Уникальный идентификатор типа события.


**evt:Тип события**
:   Определение класса событий и их метаданных.


**evt:Схема события**
:   Машиночитаемое описание структуры полей (ключи и типы значений) типа события.


**evt:Политика атрибуции**
:   Требование (или его отсутствие) наличия в рамках данного решения уже произошедшего
    другого события. Например, событие "Клик" требует событие "Показ".


**evt:Статус**
:   Статус обработки события может принимать одно из следующих значений:

    - ожидает атрибуцию
    - принят


## Метрики


**stat:ID Метрики**
:   Уникальный идентификатор метрики.

**stat:Метрика**
:   Определение метрики — числового показателя, вычисляемого посредством агрегации
    определённых событий.


## Guardrail


**guard:ID Правила**
:   Уникальный идентификатор правила.


**guard:Правило**
:   Определение метрики, окна наблюдения, порога срабатывания и результирующего действия,
    необходимого для предотвращения деградации вследствие эксперимента.


**guard:Журнал аудита**
:   Совокупность записей аудита.


**guard:ID Записи**
:   Уникальный идентификатор записи в журнале аудита.


**guard:Запись аудита**
:   Содержит подробную информацию о срабатывании <guard:Правило|Правила> для
    дальнейшего анализа причин провала.


## Продвинутые механики

**conf:Конфликтный домен**
:   Логическая область продукта (например, "Оформление заказа"), в которой 
    одновременно может быть активен только один эксперимент для пользователя.

**ramp:Ramp-план**
:   Конфигурация автоматического увеличения доли аудитории эксперимента 
    при соблюдении условий безопасности.

**lib:Learning**
:   Структурированная запись об итогах эксперимента в базе знаний.
