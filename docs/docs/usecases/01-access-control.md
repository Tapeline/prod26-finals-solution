## Сценарии

### Создание пользователя

**Кто**: <access:Администратор>

**Шаги**:

1. Генерация <access:ID>
2. Создаёт пользователя
3. Задаёт сразу его роль
4. <access:IAP ID> пользователя = null

### Активация пользователя

**Кто**: любой пользователь

**Шаги**:

1. Система ассоциирует <access:IAP ID> пользователя с внутренним <access:ID>.

!!! see-also
    Процесс определён в [ADR001](../adr/001-auth.md)

### Изменение пользователя

**Кто**: <access:Администратор>

**Шаги**:

1. Изменяет email пользователя и его <access:Роль>

### Установка правил ревью для Экспериментатора

**Кто**: <access:Администратор>

**Шаги**:

1. Задаёт список <access:ID> <access:Утверждающий|Утверждающих> и порог утверждения

### Создание первого администратора

**Кто**: Владелец системы

**Шаги**:

С помощью внутреннего, закрытого эндпоинта создаёт пользователя по
такому же флоу, как и в описанном ранее юзкейсе создания пользователя.
