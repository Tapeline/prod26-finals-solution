# Отчётность

## Сущности

### Метрика

{{ decorated_definition("stat", "Метрика") }}

- <stat:ID Метрики>
- формула по [DSL](../spec/metric-dsl.md)

### Отчёт

{{ decorated_definition("stat", "Отчёт") }}

- <stat:ID Отчёта>
- набор <stat:Метрика|Метрик> для формирования отчёта
- фиксированное временное окно: даты `[начало; конец)`
- <exp:ID> связанного эксперимента

## Сценарии

### Добавление метрики в каталог

**Кто**: <access:Экспериментатор>, <access:Администратор>

**Ограничения**:

- такого <stat:ID Метрики> не существует

**Шаги**:

1. Добавить с указанными данными

### Просмотр каталога метрик

**Кто**: любой пользователь

### Просмотр конкретной метрики

**Кто**: любой пользователь

### Обновление метрики

**Кто**: <access:Экспериментатор>, <access:Администратор>

**Шаги**:

1. Изменить формулу расчёта на новую

### Создание отчёта

**Кто**: <access:Экспериментатор>, <access:Администратор>, <access:Утверждающий>

**Ограничения**:

- дата начала не должна быть позже даты конца

**Шаги**:

1. Создаётся отчёт по выбранным данным

### Удаление отчёта

**Кто**: <access:Экспериментатор>, <access:Администратор>, <access:Утверждающий>

**Шаги**:

1. Удаляется выбранный отчёт

### Запрос отчёта

**Кто**: любой пользователь

**Шаги**:

1. Запрашиваются все определения затронутых типов событий
2. Для каждой выбранной метрики в отчёте

    1. Компилируется запрос на основании параметров метрики и типа поля из типа события
    2. Выполняется запрос к хранилищу событий в соответствии с настройкой метрики

3. Возвращаются данные по каждой метрике:

    - по эксперименту в целом
    - по каждому варианту в частности

### Запрос списка отчётов по эксперименту

**Кто**: любой пользователь

**Шаги**:

1. Возвращается список созданных для эксперимента отчётов
