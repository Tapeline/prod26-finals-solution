## Сценарии

### Создание правила оповещения

**Кто**: <access:Администратор>, <access:Экспериментатор>, <access:Утверждающий>

1. Проверяется валидность триггера и строки подключения
2. Создаётся правило оповещения с заданными данными

### Просмотр списка правил оповещения

**Кто**: любой пользователь

### Изменение правила оповещения

**Кто**: <access:Администратор>, <access:Экспериментатор>, <access:Утверждающий>

1. Проверяется валидность триггера и строки подключения
2. Изменяется правило оповещения с заданными данными

### Удаление правила оповещения

**Кто**: <access:Администратор>, <access:Экспериментатор>, <access:Утверждающий>

1. Удаляются все связанные Подготовленные оповещения
2. Удаляется правило

### Публикация оповещения

**Кто**: вызывается в коде системой, сценарий является функцией,
    не имеет внешних точек входа.

**Шаги**:

1. Высчитывается <notif:Отпечаток> (См. [ADR004](../adr/004-notification-grouping.md))
2. Если такой Отпечаток уже есть, обновляются метаданные и время
3. Иначе создаётся новое Подготовленное оповещение

### Периодическая отправка

**Кто**: фоновый процесс (воркер), раз в N секунд (определяется конфигурацией)

**Шаги**:

1. Получить Подготовленные оповещения
2. Сгруппировать по ID Правила
3. Для каждого правила и группы оповещений по нему:
    
   1. Проверить время последней высылки по этому правилу
   2. Если не прошло достаточно времени, то пропускаем группу
   3. Иначе форматируем каждое сообщение и отправляем, затем вычёркиваем из Подготовленных
