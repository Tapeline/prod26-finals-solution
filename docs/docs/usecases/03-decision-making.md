## Сценарии

### Получение значения флагов

**Кто**: Субъект

**Шаги**:

1. Получить список активных экспериментов
2. Приклеить к нему список Экспериментов, остановленных по соображениям безопасности
3. Получить список уже принятых решений по этому субъекту для
   запрошенных им флагов. 
4. Получить текущее состояние кулдауна для субъекта:
   
    - Если в кулдауне, сразу перейти к п. 8.
    - Иначе продолжить

5. Отбросить эксперименты, не имеющие отношения к оставшимся флагам
6. Сгруппировать эксперименты, имеющие определение Конфликтного домена, 
   по Конфликтным доменам, затем для каждой группы:
   
    1. Если эксперимент в группе один, оставить его, перейти к следующей группе
    2. Иначе если хотя бы у одного эксперимента Политика разрешения конфликтов
       = "Один или никто", то отбросить все эксперименты из этой группы, создав
       по каждому из них запись о разрешении конфликта.
    3. Иначе если у всех Экспериментов Политика разрешения конфликта 
       = "Наивысший приоритет", то отсортировать в порядке возрастания числового
       значения приоритета, отбросив эксперименты с незаданным приоритетом.
       Взять самый первый (наименьшее числовое значение приоритета) Эксперимент.
       Если таковых несколько, использовать детерминированный tie-breaker (
       например, на основе ID экспериментов). Полученный эксперимент считается 
       "победителем", остальные — "проигравшими". Все "проигравшие" отбрасываются,
       и по каждому эксперименту создаётся запись о разрешении конфликта
       (в т.ч. и на "победителя").

7. Для каждого из оставшихся экспериментов:

    - Если задано правило таргетирования и оно выполнилось ИЛИ
      если не задано правило таргетирования, то:
       
        1. определить Вариант значения, согласно алгоритму, 
           описанному в [ADR002](../adr/002-decision.md). НО ЕСЛИ
           эксперимент имеет статус остановленного по соображениям
           безопасности, то сразу выбрать контрольный вариант, минуя
           распределение.
        2. записать Решение, принятое по субъекту-флагу-эксперименту.
        3. исключить этот флаг из рассмотрения

8. Для всех оставшихся нерассмотренных флагов:

    - Если удалось получить значение по умолчанию, принять решение показать
      значение по умолчанию.
    - Иначе не принимать решение по этому флагу.

9. Сохранить все Записи о разрешении конфликтов
10. Сохранить все вновь принятые решения (кроме вызванных остановкой по
соображениям безопасности).
11. Вернуть отображение Флаг -> Принятое решение. При отсутствии решения
отобразить Флаг в null.

    - [ADR002 — Алгоритм принятия решений по флагам](../adr/002-decision.md)
    - [ADR007 — Алгоритм разрешения конфликтов](../adr/007-conflict-resolution.md)

!!! note
    Получение значений по умолчанию и текущих запущенных и остановленных
    по соображениям безопасности экспериментов происходит из кэша.

    Получение уже принятых решений, состояния кулдауна и сохранение решений
    и Записей о разрешении влёчет накладные расходы на поход в базу.

    Это известное ограничение.
