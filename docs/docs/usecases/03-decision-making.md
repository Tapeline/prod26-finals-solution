**Ограничения**:
- **Детерминизм**: при той же конфигурации один и тот же Субъект получает тот же результат.
- **Stickiness**: пользователь не должен менять вариант внутри одного эксперимента.
- **Защита пользователя**: 
    - Ограничение количества одновременных экспериментов на одного субъекта. ??? как именно выбираются приоритетные ???
    - Соблюдение периодов "отдыха" (cooldown). ??? глобальный или по доменам ???

**Шаги**:
1. Система получает ID субъекта, его атрибуты и список ключей флагов.
2. Для каждого флага:
    1. Проверяет наличие активного (Запущен) эксперимента.
    2. Если эксперимента нет или он на паузе — возвращает значение по умолчанию.
    3. Если эксперимент есть:
        1. Проверяет <exp:Правило таргетирования> по атрибутам субъекта.
        2. Проверяет политики защиты (квоты, cooldown).
        3. Если проверки не пройдены — возвращает значение по умолчанию.
        4. Если пройдены — детерминированно (через хеш от ID субъекта и соли эксперимента) определяет <exp:Вариант>.
3. Генерирует <rt:Decision ID> для связки с будущими событиями.
4. Возвращает результат.
