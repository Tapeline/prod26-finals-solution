## Сценарии

### Получение значения флагов

??? главный вопрос ???

**Кто**: Субъект

**Шаги**:

1. Система получает ID субъекта, его атрибуты и список ключей флагов.
2. Для каждого флага:
    1. Проверяет наличие активного (Запущен) эксперимента.
    2. Если эксперимента нет или он на паузе — возвращает значение по умолчанию.
    3. Если эксперимент остановлен по соображениям безопасность — возвращает контрольный вариант.
    4. Если эксперимент есть:
        1. Проверяет <exp:Правило таргетирования> по атрибутам субъекта.
        2. Добавляет эксперимент в список ожидающих
3. Если есть конфликты в списке ожидающих экспериментов, то решить их
   для каждого домена, как описано в [ADR007](../adr/007-conflict-resolution.md).
   ??? решать до политик защиты пользователя или после ???
4. Для всех оставшихся после конфликтов экспериментов:
    1. Проверяет политики защиты (квоты, cooldown).
    2. Если проверки не пройдены — возвращает значение по умолчанию.
    3. Если пройдены — детерминированно определяет <exp:Вариант>,
       как описано в [ADR002](../adr/002-decision.md).
5. Генерирует <rt:ID Решения> для связки с будущими событиями.
6. Сохраняет <conflict:Запись о разрешении|Записи о разрешении конфликтов>.
7. Возвращает результат.
