### Архивировать тип события

**Кто**: <access:Экспериментатор>, <access:Администратор>

**Шаги**:

1. Пометить как архивированный

### Отправка событий

**Кто**: Субъект

**Ограничения**:

- <evt:Тип события> существует
- payload соответствует схеме типа события

**Шаги**:

1. Для каждого события:

   1. Если такого <evt:Тип события> не существует, пометить как невалидное, конец.
   2. Если payload не соответствует схеме типа события, пометить как невалидное, конец.
   3. Если такой <evt:ID> уже зарегистрирован, пометить как дубликат, конец.
   4. Если в рамках данного <rt:ID Решения> есть событие, подходящее под Политику
      атрибуции этого события:
   
        1. Пометить это как принятое
        2. Пометить рекурсивно все остальные события, ожидающие это по Политике атрибуции.

   5. Иначе пометить как ожидающее атрибуцию

2. Вернуть отчёт по всему батчу событий
