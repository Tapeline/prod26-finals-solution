# Карта репозитория

## `alphabet-backend`

### `scripts`

Вспомогательные скрипты для разработки.

- `pkgs.py` — скрипт для получения списка всех пакетов в проекте. Нужен для
  удобного использования [`arch_blueprint`](https://github.com/nkhitrov/arch_blueprint)

### `src/alembic`

Конфигурация миграций, файлы для их генерации и сами миграции:

- `env.py` — конфигурация
- `versions/*` — миграции
- `script.py.mako` — шаблон миграции

### `src/alphabet/**`

Модули приложения. Каждый модуль подчиняется правилам Чистой архитектуры:

- `application` — слой прикладной логики (варианты использования и порты)
- `domain` — слой критической логики
- `infrastructure` — слой инфраструктуры (адаптеры)
- `presentation` — слой представления (http-контроллеры)

Между модулями не должно быть циклических зависимостей. Эти правила проверяются
с помощью `import-linter`.

### `src/clickhouse_init.sql`

Файл инициализации таблиц в ClickHouse.

### `src/cooldown.lua`

Redis-скрипт, реализующий логику кулдаунов.

### `static`, `templates`

Файлы фронтенда (JS, CSS, HTML).

### `tests`

Юнит- и интеграционные тесты сервиса.

### `alphabet.yml`

Файл конфигурации сервиса.

### Конфигурации инструментов

- `.coveragerc` — конфигурация Code Coverage
- `.flake8`, `ruff.toml` — конфигурация линтеров
- `justfile` — точка входа для инструментов (аналог Makefile)
- `mypy.ini` — конфигурация тайпчекера.
- `.importlinter` — конфигурация архитектурных тестов
- `start_*.sh` — точки входа для контейнеров

## `docs`

Документация по проекту.

- `docs` — md и другие файлы, непосредственно относящиеся к контенту документации
- `macros.py` — mkdocs-макросы документации
- `mkdocs.yml` — конфигурация документации

## `dummy-iap`

IAP-заглушка, необходимый для проверки отдельной функциональности 
(управление аккаунтами, Insights UI).

- `app.py` — основной файл IAP-сервиса
- `static` — JS и CSS файлы IAP-сервиса
- `templates` — Jinja-шаблоны страниц IAP-сервиса
- `config.toml` — конфигурация IAP. Здесь же указываются доступные пользователи.

## `sys-tests`

Приёмочные и end-to-end тесты.

### `tests/bXX` / `tests/fXX`

Файлы приёмочных и end-to-end тестов, сгруппированные по критериям.

### `tests/scenario`

Файлы тестов, прогоняющих полный тестовый сценарий.

### Вспомогательные `tests/` файлы

`client.py`, `config.py`, `conftest.py`, `helpers.py` определяют вспомогательные
функции и pytest-фикстуры, необходимые для выполнения тестов

### Конфигурации

- `alphabet.test.yml` — конфиг-файл основного сервиса со значениями, необходимыми
  для тестирования в рамках E2E тестов
- `config.toml` — конфиг-файл тестов

## Файлы в корне

- `docker-compose.autotest.yml` — Compose-файл для автотестов в CI
- `docker-compose.yml` — Основной compose-файл
- `OLD_README.md` — изначальный README.md, данный организаторами
