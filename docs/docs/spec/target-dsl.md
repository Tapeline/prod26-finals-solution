# DSL таргетирования

## Определение грамматики

```ebnf

expr ::=
    disj
    
disj ::=
    conj { "OR" conj }

conj ::=
    cmp { "AND" cmp }

cmp ::=
    unary { ("IN" | "NOT IN" | ">" | ">=" | "<" | "<=" | "==" | "!=") unary }

unary ::=
    { "NOT" } primary

primary ::=
    "(" expr ")" |
    collection |
    name |
    value

value ::=
    date |
    string |
    number |
    bool |
    "undefined"
    
collection ::= 
    "[" [ value ] { "," value } "]"

date ::=
    regex [0-9]{4}-[0-9]{2}-[0-9]{2}

string ::=
    string literal in double quotes with \n, \t, \r, \v, \" and \\ escape sequences support

number ::=
    regex -?[0-9]+(\.[0-9]+)?

name ::=
    regex [a-zA-Z_][a-zA-Z0-9-_]*

bool ::=
    true | 
    false

```

## Правила вычисления

При любой возникшей ошибке вычисления, выражение становится равно `false`.

Строки сравниваются лексикографически.

Даты сравниваются по годам, потом по месяцам, потом по дням.

Числа сравниваются с допустимой погрешностью 1e-9 (известная проблема 0.1 + 0.2 != 0.3).

Значения разных типов не равны между собой ни при каких обстоятельствах.

`>`, `<`, `>=`, `<=` сравнения неподдерживаемых типов приводят к ошибкам.

Любая операция с `undefined` результирует в `false`, кроме:

- `undefined or some_value -> some_value`.
- `NOT undefined -> true`

NOT имеет такую же семантику, как и в Python.

## Неявное приведение (coercion)

Если литерал даты сравнивается с именем, содержащим в себе строку, то:

1. Производится попытка приведения строки к дате вида YYYY-mm-dd
2. Производится попытка приведения строки к дате вида ISO format
3. При ошибке, сравнение возвращает `false`

Если литерал даты сравнивается с именем, содержащим в себе число, то:

1. Производится попытка приведения строки к дате с помощью UTC unix timestamp
2. При ошибке, сравнение возвращает `false`

Если оба сравниваемых значения — строки, соответствующие паттерну `\d+\.\d+\.\d+`,
то они приводятся к семантической версии и сравниваются покомпонентно.

!!! note
    Coercion включен на операциях: `>`, `<`, `>=`, `<=`, `==`, `!=`
