# Описания выборки метрик

Каждая метрика имеет:

- фильтр по типу события
- адрес поля со значением (в дополнительных параметрах события)
- функцию агрегации по этим значениям

## Фильтр на события

Описывается строкой, содержащей в себе один <evt:ID Типа> события

## Адрес поля

Описывается строкой, содержащей в себе путь до поля в таком формате:

```ebnf

address ::=
    special |
    key {  "." key  |  ":" index  }

special ::=
    "!event_latency"

key ::=
    regex [A-Za-z0-9_-]+

index ::=
    regex [0-9]+

```

??? переделать, чтобы поддерживать конверсии!


Например, чтобы достать до "Apple" в этом сообщении:

```json
{
  "type": "purchase",
  "items": [
    {"name": "Apple"}
  ]
}
```

Нужно написать: `items:1.name` (нумерация начинается с 1!)

## Функция агрегации

Вычисляет значение метрики. Доступны следующие функции:

- `rel` — Отношение
- `avg` — Среднее арифметическое
- `count` — Кол-во значений
- `sum` — Сумма значений
- `p50` — 50-й перцентиль
- `p75` — 75-й перцентиль
- `p90` — 90-й перцентиль
- `p95` — 95-й перцентиль
- `p99` — 99-й перцентиль
- `max` — Максимальное значение
- `min` — Минимальное значение
