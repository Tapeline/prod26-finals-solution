# Собственный DSL для расчёта метрик

## Контекст и проблема

Необходимо поддерживать настраиваемые метрики, например:

- конверсии
- перцентили задержки
- количество ошибок
- долю ошибок
- и так далее

Это требует крайне гибкого инструмента задания метрик.

## Рассмотренные варианты

- сырые clickhouse выражения
- собственный DSL

## Решение

Принято решение использовать собственный DSL

### Последствия

- Хорошо, потому что сильно безопаснее, чем сырые выражения
- Плохо, потому что требует много времени на реализацию

## "За" и "против"

### Сырые clickhouse выражения

- Хорошо, потому что быстро реализовать
- Плохо, потому что дыра в безопасности (можно выполнить DROP TABLE, например)
