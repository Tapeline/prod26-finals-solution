# Insights UI как легковесный фронтенд

## Контекст и проблема

Требуется иметь возможность смотреть статистику по эксперименту в
графическом интерфейсе. Она должна отражать, как следует из ТЗ:

- долю отклонённых событий и причины (топ причин);
- долю дубликатов;
- долю событий конверсии/ошибок/задержки, которые временно не дождались экспозиции (или наоборот — степень полноты атрибуции);
- задержку доставки событий — если собирается;
- перекос трафика: фактическое распределение показов по вариантам против ожидаемого по весам (как предупреждение).

Необходимо иметь способ получения визуальной статистики + отчёта в CSV.

## Рассмотренные варианты

- SSR в PNG (matplotlib и т.д.)
- SSR в HTML 
- API + лёгкий фронтенд (fetch)

## Решение

Выбран API + лёгкий фронтенд (fetch), так как он обеспечивает лучшее время отклика
и позволяет расширять функциональность в дальнейшем.

### Последствия

- Хорошо, потому что будет низкое время отклика (нужно лишь запросить новые данные, не надо
  отрисовывать всё с нуля в PNG или HTML)
- Хорошо, потому что наличие разделения на API и фронтенд для отрисовки позволит 
  платформе проще расширяться (улучшать фронтенд, добавлять новые метрики)
- Плохо, потому что требует дополнительных затрат на реализацию фронтенда

### Реализация

Фронтенд делает запрос на эндпоинт статистики (с выбранными или нет фильтрами разреза),
затем отрисовывает статистику с помощью, например, ApexCharts или Apache ECharts.

Фильтр разреза (а значит и данные) зависят от GET параметров, так что для того,
чтобы поделиться графиком, достаточно будет скопировать полную ссылку.

На фронтенде будут отображаться:

- pie chart по статусам событий (отклонён/дубликат/ожидает атрибуцию/принят)
- pie chart по типам событий
- pie chart по причинам отклонения
- gauge полноты атрибуции (`(принято / (принято + ожидают атрибуцию)) * 100%`)
- p50, p75, p95 задержки доставки
- pie chart реального распределения показов по вариантам
- таблица отклонений от ожидаемого распределения по весам
- предупреждение о слишком большом отклонении в распределении

Экспорт CSV будет отдавать ZIP файл с такими CSV:

- CSV причин отклонения:

    | Причина | Кол-во отклонённых |
    |---------|--------------------|
    |         |                    |

- CSV по вариантам

    | Вариант | Сколько пользователей получило | Ожидаемая доля распределения | Реальное распределение | Сколько событий отклонено | Сколько событий принято |
    |---------|--------------------------------|------------------------------|------------------------|---------------------------|-------------------------|
    |         |                                |                              |                        |                           |                         |

- CSV по типам событий

    | Тип события | Сколько пришло |
    |-------------|----------------|
    |             |                |


## "За" и "против"

### SSR в PNG

- Хорошо, потому что очень просто реализовать
- Плохо, потому что полностью отсутствует интерактивность
- Плохо, потому что долго переотрисовывается
- Плохо, потому что бэкенд нагружен отрисовкой

### SSR в HTML

- Хорошо, потому что проще, чем API + фронтенд
- Плохо, потому что долго переотрисовывается
- Плохо, потому что бэкенд нагружен отрисовкой
